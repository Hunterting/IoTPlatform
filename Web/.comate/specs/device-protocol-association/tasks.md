# 协议配置设备关联功能实现任务

## 任务概述
为协议配置页面添加设备关联和查看功能，使管理员能够清楚地知道每个协议接入了哪些设备，并能够进行设备关联管理。

- [x] 任务 1：扩展 ProtocolConfig 接口定义
    - 1.1: 在 DataCollectionPage.tsx 中扩展 ProtocolConfig 接口，添加 deviceIds: string[] 字段
    - 1.2: 更新初始数据，为示例协议添加 deviceIds 关联数据
    - 1.3: 确保 deviceCount 字段根据 deviceIds 数组长度动态计算

- [x] 任务 2：实现协议卡片设备列表展开功能
    - 2.1: 在协议卡片底部添加"查看关联设备"按钮
    - 2.2: 实现 getProtocolDevices 函数，根据 deviceIds 过滤设备列表
    - 2.3: 添加展开/收起状态管理（expandedProtocols Set）
    - 2.4: 实现设备列表展开动画和展示逻辑
    - 2.5: 在展开列表中显示设备名称、序列号、在线状态

- [x] 任务 3：实现从协议移除设备功能
    - 3.1: 在设备列表项中添加移除按钮
    - 3.2: 实现 handleRemoveDevice 函数，更新协议的 deviceIds
    - 3.3: 自动更新 deviceCount 计数
    - 3.4: 添加移除确认提示

- [x] 任务 4：实现协议配置弹窗中的设备选择器
    - 4.1: 在协议配置表单底部添加设备关联区域
    - 4.2: 实现已选设备标签展示（显示设备名称和移除按钮）
    - 4.3: 添加"选择设备"按钮，触发设备选择器弹窗
    - 4.4: 实现 editingProtocolDevices 状态管理

- [x] 任务 5：创建独立的设备选择器组件
    - 5.1: 创建 DeviceSelectorModal 组件，支持设备搜索和状态筛选
    - 5.2: 实现设备列表展示，包含复选框、设备名称、序列号、状态
    - 5.3: 实现设备选中/取消选中逻辑
    - 5.4: 添加确认和取消按钮，显示选中设备数量
    - 5.5: 添加滚动支持和最大高度限制

- [x] 任务 6：更新协议编辑和保存逻辑
    - 6.1: 修改 handleEdit 函数，加载协议的 deviceIds 到 editingProtocolDevices
    - 6.2: 修改 handleAdd 函数，清空 editingProtocolDevices
    - 6.3: 修改 handleSave 函数，保存 editingProtocolDevices 到协议的 deviceIds
    - 6.4: 确保 deviceCount 在保存时正确计算

- [x] 任务 7：优化UI/UX体验
    - 7.1: 添加展开/收起图标动画效果
    - 7.2: 优化设备列表的视觉样式和间距
    - 7.3: 为空设备列表添加友好提示
    - 7.4: 优化设备选择器的搜索和筛选交互
